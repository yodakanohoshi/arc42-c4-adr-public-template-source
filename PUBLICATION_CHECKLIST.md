# Public repository publication checklist

公開前に、少なくとも次を確認してください。

## 固有情報・機密情報

- [ ] 社名、部署名、個人名、メールアドレスが残っていない
- [ ] 実在するドメイン、URL、IPアドレス、アカウントIDが残っていない
- [ ] APIキー、Secret、Cookie、証明書、資格情報が含まれていない
- [ ] 実際のシステム名、顧客名、プロジェクト名が含まれていない
- [ ] 実データ、ログ、スクリーンショットが含まれていない
- [ ] 生成済みPDF・HTML・画像のメタデータも確認した

推奨スキャン例:

```bash
rg -n -i 'password|secret|token|api[-_ ]?key|private[-_ ]?key' . 
git log --all --stat
```

専用のSecret Scannerを組織標準に従って併用してください。

## テンプレート固有

- [ ] `[記入]`、`[リンク]`、`YYYY-MM-DD`の残存が意図したものか確認した
- [ ] サンプル値を実際のSLOや契約条件と誤認しない表示になっている
- [ ] `mkdocs.yml` の公開URLを設定した
- [ ] `pdf/config.yml` のタイトル、著者、ヘッダーを確認した
- [ ] `LICENSES/MIT.txt` の著作権表示を更新した
- [ ] `NOTICE.md` の帰属を維持した

## ビルド・品質

- [ ] `docker compose run --rm --build verify` が成功する
- [ ] Webサイトのリンク切れがない
- [ ] PDFの表、図、コードが欠けていない
- [ ] 目次と章順が正しい
- [ ] 不要な `site/`、`build/`、一時ファイルをコミットしていない

## Git履歴

過去のコミットに機密情報が含まれる場合、現在のファイルを削除するだけでは不十分です。公開前に履歴全体を確認し、組織の手順に従って履歴を書き換えてください。
