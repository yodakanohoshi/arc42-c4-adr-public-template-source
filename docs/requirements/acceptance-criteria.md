# 受入条件テンプレート

## シナリオ形式

```text
シナリオID: AC-CORE-001
関連要件: REQ-FUNC-001

前提:
- 利用者が必要な操作権限を持つ
- 必須の参照データが利用可能である

操作:
- 利用者が必要な情報を入力し、処理を実行する

期待結果:
- 入力が検証される
- 業務データが一度だけ更新される
- 処理結果が表示される
- 監査イベントが記録される
```

## Given / When / Then例

```gherkin
Scenario: 同じ要求を再送しても二重更新されない
  Given 一意な要求IDを持つ処理が正常に完了している
  When 同じ要求IDで処理を再送する
  Then 業務データは二重に更新されない
  And 最初の処理結果と整合する応答が返る
  And 重複要求として記録される
```
