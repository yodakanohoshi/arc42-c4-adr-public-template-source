# 4. ソリューション戦略

詳細な実装ではなく、品質目標を実現するための上位方針を1〜2ページで説明する。

## 4.1 主要戦略

| 戦略ID | 品質目標 | 戦略 | 関連ADR |
|---|---|---|---|
| STR-001 | 変更容易性 | 責務とデータ所有を明確に分離する | ADR-0001 |
| STR-002 | 相互運用性 | 外部インターフェースを標準化し、契約を機械可読にする | [ADR-0001](adr/0001-machine-readable-interface-contract.md) |
| STR-003 | 信頼性 | 障害領域を分離し、重要経路の単一障害点を避ける | ADR-0002 |
| STR-004 | 回復性 | 非同期処理は冪等性、再試行、失敗隔離を備える | ADR-0003 |
| STR-005 | 運用性 | Logs、Metrics、Tracesを相関可能にする | ADR-0004 |
| STR-006 | 再現性 | 環境と設定を自動化された定義から再構築できるようにする | ADR-0005 |
| STR-007 | セキュリティ | 最小権限、秘密情報の外部管理、監査証跡を共通原則とする | ADR-0006 |

## 4.2 提案・検討が必要な事項

- 実行基盤とデータストアの選定
- 同期・非同期インターフェースの使い分け
- 障害時の縮退動作
- データ整合性とトランザクション境界
- バックアップ、復旧、災害対策方式
- デプロイ、ロールバック、互換性維持の方式

重要な提案は[Design Doc / RFC](supporting-templates/design-doc.md)で比較し、確定後にADRへ記録する。
