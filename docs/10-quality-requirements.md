# 10. 品質要件

「高可用」「安全」「高速」といった形容詞だけでなく、測定・検証可能なシナリオとして記載する。以下の値は例であり、対象システムの要求へ置き換える。

## 10.1 品質要件一覧

| ID | 品質 | 要件例 | 検証方法 | 優先度 |
|---|---|---|---|---|
| Q-REL-001 | 信頼性 | 主要処理の成功率を合意した目標以上とする | SLI集計 | Must |
| Q-PERF-001 | 性能 | 通常負荷時の主要API応答p95を合意値以内とする | 負荷試験・監視 | Must |
| Q-REC-001 | 回復性 | 単一障害領域の停止で重要機能全体を停止させない | 障害試験 | Must |
| Q-DATA-001 | データ保護 | 合意したRPOとRTOを満たす | 復旧訓練 | Must |
| Q-SEC-001 | セキュリティ | 管理操作を実行者、日時、対象、結果とともに追跡できる | セキュリティ試験 | Must |
| Q-CHG-001 | 変更容易性 | 標準的な変更を合意したリードタイムで安全にリリースできる | 演習・実績 | Should |
| Q-OPS-001 | 運用性 | 重大な異常を合意した時間内に検知できる | Alert試験 | Must |
| Q-SCALE-001 | 拡張性 | 想定ピーク負荷で処理量を維持できる | 負荷試験 | Should |

## 10.2 品質シナリオ形式

| 項目 | 内容 |
|---|---|
| Source | 誰・何が刺激を発生させるか |
| Stimulus | 何が起きるか |
| Environment | 通常時、ピーク、障害時など |
| Artifact | 対象となる要素 |
| Response | システムがどう応答するか |
| Measure | 時間、割合、件数などの判定値 |

## 10.3 SLOとの関係

- 第10章: 利用者・事業・契約として必要な品質
- SLO台帳: 運用中に測定するSLIと目標
- 監視台帳: 収集方法、Dashboard、Alert、Owner
- 試験仕様: リリース前に品質を検証する方法

未確定の数値には、決定者、決定期限、見積時の暫定値を付ける。
