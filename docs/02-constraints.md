# 2. アーキテクチャ制約

設計上の自由を制限する、変更困難な条件を記載する。設計案や希望事項を必須制約へ混ぜない。

## 2.2 文書・納品制約

- 図は画像だけでなく編集可能な元データも保持する
- 設定値や一覧は機械可読または編集可能な形式で管理する
- 文書ごとにOwner、状態、最終レビュー日を付与する
- PDFのみを正の情報源にせず、更新可能な原本を保持する
- 秘密情報や実データを設計文書へ埋め込まない

## 2.3 制約・品質要件・設計判断の区別

| 記述例 | 分類 |
|---|---|
| 指定された実行環境を利用しなければならない | 制約 |
| 特定の実行方式を採用する | 設計判断。ADR候補 |
| 月間可用性を規定値以上とする | 品質要件 |
| 複数の障害領域へ冗長配置する | 品質要件を満たすための設計戦略 |
